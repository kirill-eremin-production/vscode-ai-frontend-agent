## Context

После #0017 composer создания рана убран из левой панели и должен жить в main-area как отдельный экран. Параллельно в main-area нужен осмысленный empty state — то, что видит пользователь, у которого ещё нет ранов или ничего не выбрано.

Сейчас при отсутствии выбранного рана main-area либо пустая, либо показывает «выберите ран» — это выглядит как поломка, а не как осмысленное «начало работы».

## Acceptance criteria

- **Empty state** (`mainAreaMode === 'empty'`, `selectedRunId === undefined`):
  - Используется атом `EmptyState` (#0016): крупная lucide-иконка («Sparkles» / «Bot»), заголовок «Запустите команду агентов», короткое описание роли расширения (1–2 предложения), primary-кнопка «Новый ран». Кнопка вызывает то же действие `startNewRun()`, что и кнопка из шапки списка.
- **Форма создания** (`mainAreaMode === 'new-run'`):
  - Заголовок экрана «Новый ран».
  - Поле «Заголовок (необязательно)» — single-line input. Если оставлено пустым — заголовок генерится моделью (US-2, существующая логика).
  - Поле «Запрос» — textarea минимум 8 строк, autosize до разумного потолка. Placeholder с примером запроса.
  - Кнопка «Запустить» (primary, из #0016) — disabled при пустом запросе, с tooltip-объяснением. На время IPC-запроса показывает `loading` с подписью «Создаю ран…».
  - Кнопка «Отмена» (ghost) — возвращает в empty state (или к ранее выбранному рану, если он был).
  - Если ключа OpenRouter нет — на «Запустить» система триггерит существующий flow ввода ключа (US-1); после успешного ввода — продолжает создание рана без потери введённого текста.
- После успешного `runs.create`:
  - main-area переключается в режим `run-details` со свежесозданным `runId`.
  - В левой панели новый ран появляется и подсвечен.
  - Введённые в форме данные очищаются (повторное «+ Новый ран» начинает с пустого).
- Если `runs.create` упал (ошибка ключа/сети/чего угодно):
  - Форма остаётся открытой, текст пользователя не теряется.
  - Под кнопкой — error-сообщение с понятным текстом и кнопкой «Повторить».
- Поведение без открытого workspace (US-5): на кнопке «Новый ран» (и в empty state, и в шапке списка) — disabled с tooltip «Откройте папку проекта». Сама форма не открывается.

## Implementation notes

- Компонент `src/webview/features/new-run/ui/NewRunForm.tsx` + `EmptyState` остаются маленькими композициями из атомов #0016 — никакой собственной CSS.
- Состояние формы (заголовок/текст) — локальное в компоненте, не в глобальном store: если пользователь переключился на другой ран и вернулся, форма очистится — это сознательно, чтобы не плодить «висящие черновики». Если позже захочется черновики — отдельная задача.
- Текущий код composer'а в левой панели целиком удаляется. Не забыть перенести/переиспользовать существующие хуки валидации/IPC (`runs.create`, обработка отсутствия ключа), чтобы не дублировать.
- TC: расширить TC-31 (или новый TC) — путь «открыть webview → empty state → нажать "Новый ран" → ввести запрос → запустить → ран появился в списке и открыт». Композер в левой панели в TC должен отсутствовать (selector check).
- Без переключения тем и без сложной валидации — минимально жизнеспособный flow.
