## Context

После #0012 в `RunDetails.tsx` живёт `SessionTabs` с деревом сессий. Это работает, но смешивает навигацию по сессиям и саму ленту чата на одной поверхности. Чат должен иметь всю ширину main-area, а навигация по сессиям — лежать в собственной collapsible-панели справа (заглушку для которой ставит #0017).

Заодно сюда переезжает поведение «свёрнуто по умолчанию, если у рана одна сессия» — нет смысла занимать экран ради одного элемента.

## Acceptance criteria

- Новый компонент `src/webview/features/sessions-panel/ui/SessionsPanel.tsx` — заполняет правую панель из #0017.
- Шапка панели:
  - В развёрнутом состоянии — заголовок «Сессии», справа кнопка свернуть.
  - В свёрнутом — узкая полоска с иконкой «дерево» (lucide `Network`) и кнопкой развернуть.
- Тело — дерево сессий (логика рендера полностью переезжает из текущего `SessionTabs` в `RunDetails.tsx`):
  - Группировка по `parentSessionId`, отступ по глубине (как сейчас).
  - Маркер `●` для активной (live) сессии.
  - Бейдж `kind` («🤝 Передача» для `agent-agent`, «Чат N» для `user-agent`) — через атом `Badge` из #0016.
  - Сортировка корней по `createdAt` сохраняется.
- Клик по сессии — переключает просматриваемую сессию в чате (`store.selectSession(runId, sessionId)`), как сейчас. Поведение active vs read-only сессии (composer disabled с подсказкой) — без изменений.
- Если у рана **одна** сессия — панель **по умолчанию свёрнута**, в её свёрнутой шапке отображается «1 сессия» (микро-бейдж рядом с иконкой).
- Если сессий **больше одной** — панель **по умолчанию развёрнута** при первом открытии этого рана. Дальнейшие изменения collapsed-состояния пользователем — persist'ятся per-run в `globalState` (ключ `sessionsPanel.collapsed.<runId>`).
- Из `RunDetails.tsx` удаляются: `SessionTabs`, `buildSessionTree`, `SessionTreeLevel` — и вся связанная логика. Никаких параллельных копий быть не должно.

## Implementation notes

- Логика дерева перевозится 1-в-1, но обёрнута в новый layout. Если в процессе захочется сразу починить какую-то нестыковку — не делать в этой задаче, заводить отдельный issue, иначе размер вырастет.
- Per-run persist через тот же generic IPC `state.setUiPref` из #0017. Ключ — с префиксом `runId`, чтобы не плодить отдельные хэндлеры.
- При переключении рана панель пересчитывает свой default (одна сессия → свёрнута, больше → развёрнута), если для нового рана в `globalState` ничего не сохранено. Если сохранено — берём из стораджа.
- Тесты: TC-31 и TC-32 проверяют поведение чата по сессиям; никаких regress'ов быть не должно. Селекторы дерева в e2e обновятся (новый путь рендера) — поправить в тестовых хелперах.
- Аватары/иконки ролей в дереве пока опциональны — основной компонент дерева можно оставить минималистичным; визуал прокачаем при работе над канвасом, если станет тесно.
