## Context

После #0023–#0025 канвас живёт и общается. Не хватает главного: возможности из карты в один клик «провалиться» внутрь конкретного диалога — открыть чат агента, открыть конкретную bridge-сессию, посмотреть всю работу любого участника.

Это закрывает обещание «офис на ладони»: я смотрю на карту → вижу, что архитектор сейчас думает → кликаю по нему → читаю прямо сейчас, что он пишет.

## Acceptance criteria

- **Клик по кубику агента** на канвасе:
  - main-area переключает вкладку на «Чат».
  - В правой панели сессий (#0019) выбирается _самая свежая live_ сессия этого агента (если активная сессия рана принадлежит ему — она; иначе последняя по `updatedAt`).
  - Если у агента несколько сессий — выбирается одна (последняя), но в правой панели подсвечено «у этого агента ещё N сессий», клик по бейджу — открывает popover-список (см. ниже).
- **Клик по стрелке** между двумя агентами:
  - main-area переключает на «Чат», выбирает соответствующую bridge-сессию (та, у которой `parentSessionId` указывает на корень источника, и `participants` содержат обе роли).
  - Подсвечена в дереве сессий справа.
- **Двойной клик / shift-клик по кубику**:
  - Открывается popover (over canvas) с компактной хронологией всех сессий этого агента: список с датами и статусами, иконка `kind`. Клик по элементу popover'а — открывает выбранную сессию в чате.
  - Если popover-list одной строки (одна сессия) — поведение совпадает с обычным кликом, popover не нужен.
- **Возврат на канвас**: вкладка «Карта» в шапке main-area — всегда доступна. last-viewed `sessionId` per-run запоминается в `globalState` (`mainArea.lastSession.<runId>`), чтобы повторный заход в «Чат» открывал ту же сессию, где пользователь остановился.
- **Hover-tooltip**:
  - На кубике: имя агента, текущий статус (длинная версия из `describeRunActivity`), время последней активности, число сессий, кратко — последняя строка от него (например, «"Готов план v2"»).
  - На стрелке: тип передачи («бриф»/«план»), время handoff'а, число сообщений в bridge.
  - Tooltip — атом из #0016 (Radix), задержка ~400мс перед появлением.
- Cursor-affordance: на кубиках и стрелках — `cursor: pointer`. Drag по фону для панорамирования продолжает работать (drag начинается на пустом месте, не на кубике).
- Клавиатурная навигация (минимум): `Tab` циклит по кубикам и стрелкам с focus-ring; `Enter` — то же действие, что клик. Полноценный keyboard-mode (стрелки между кубиками) — отдельная задача.

## Implementation notes

- Действие drill-in — диспатч в store: `selectRunSession(runId, sessionId)` + `setMainAreaTab(runId, 'chat')`. Оба уже существуют (или появляются) в #0019/#0023.
- Popover-список сессий — атом `Popover` поверх `Panel` (#0016). Рекордов мало, можно без виртуализации.
- last-viewed session per-run — generic ui-pref из #0017.
- Edge → bridge-session matching: в layout #0023 каждая стрелка строится по конкретной bridge-сессии — так что у edge есть `data-session-id`, drill-in тривиален.
- Hover-tooltip с превью последнего сообщения: брать из store (`selectLastMessageOfRole(runId, role)`), truncate до ~80 символов.
- Не плодить новых IPC: всё переиспользует существующее (`runs.get` с `sessionId` уже есть с #0012).
- TC: e2e — после handoff'а кликнуть по кубику архитектора → main-area показывает «Чат» с bridge-сессией; кликнуть по стрелке → та же сессия; вернуться на «Карту» → клик «Чат» снова открывает bridge-сессию (lastSession сохранилась).
