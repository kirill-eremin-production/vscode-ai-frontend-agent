## Context

Сейчас лента в `RunDetails.tsx` рендерит сообщения как сырые блоки текста с пометкой `from`. Это читается как лог, а не как чат, и не масштабируется на длинные диалоги: непонятно, где «моё», где агентское, какое сообщение когда было.

Задача — превратить ленту в современный чат-вид с аватарами ролей, именами, относительным временем и markdown-рендерингом.

## Acceptance criteria

- Новый компонент `src/webview/features/chat/ui/ChatMessage.tsx` — одно сообщение:
  - Слева — `Avatar` (#0016) с иконкой роли по `from` (`user`, `agent:product`, `agent:architect`, `agent:system`).
  - Справа от аватара — шапка с именем («Вы», «Продакт», «Архитектор», «Система») и временем.
  - Время — относительное (`formatRelativeTime`: «только что», «2 мин назад», «вчера в 14:32», «12 апр»). Tooltip — абсолютная дата/время в локальном формате.
  - Тело — markdown-рендеринг (заголовки, списки, code-блоки, inline-code, ссылки). Code-блоки — моноширинные, с лёгким фоном, без подсветки синтаксиса (не вводим тяжёлый highlighter в этой задаче — добавим точечно при необходимости).
  - Bubble имеет `surface-elevated`-фон и тонкий бордер; для `from === 'user'` — акцентный левый бордер и чуть отличающийся фон, чтобы глазами различалось.
- Лента (`ChatFeed.tsx`):
  - Рендерит `ChatMessage` подряд, без визуальных разделителей-засечек.
  - Авто-скролл вниз при новом сообщении, если пользователь уже у дна (порог: ~64px от низа). Если выше — показывает закреплённую снизу пилюлю «↓ Новые сообщения», клик возвращает в самый низ.
  - При смене сессии (через панель из #0019) скролл сбрасывается в самый низ.
- Markdown — через `react-markdown` + `remark-gfm` (минимум). Не подключать `rehype-sanitize` — webview уже изолирован, источник markdown — наша же storage, не пользовательский HTML.
- Длинные code-блоки — с горизонтальным скроллом внутри bubble, не ломают вёрстку. Длинные слова (URL без пробелов) — `overflow-wrap: anywhere`.
- Системные пометки (например, «Продакт закончил без текста брифа…») рендерятся через `agent:system` — нейтральный аватар, чуть приглушённый стиль, не доминирует визуально.
- Tool-events в ленте остаются (как сейчас), но визуально к ним пока не лезем — это задача #0021. На время до её завершения они отображаются прежним способом, не сломаны.

## Implementation notes

- Утилита `formatRelativeTime(at: string): { label: string; tooltip: string }` — в `src/webview/shared/lib/time.ts`. Не тащить `dayjs`/`date-fns` ради одного форматирования — написать ~30 строк руками; на длинной дистанции если понадобится больше дат — заведём `date-fns` отдельно.
- Маппинг `from` → имя/аватар-иконка живёт в `src/webview/shared/ui/role-icons.ts` (создан в #0016). Здесь — только потребление.
- Тип `ChatMessage` уже существует в storage; webview-компонент не должен его дублировать — импортируем общий тип или его узкое webview-представление.
- Скроллер и логика «у дна» — `useEffect` с `ref` на контейнер; не плодить library для этого.
- TC: в существующих TC-31/32 поправить selector'ы под новые DOM-структуры; добавить точечный TC, который проверяет рендер аватара/имени/времени и наличие markdown (например, ## заголовок в брифе должен стать `<h2>`).
