## Context

Кубик роли должен показывать «что эта роль делает прямо сейчас». На этой итерации — три состояния. `paused` добавится позже в #0052.

## Acceptance criteria

- Состояния кубика:
  - `idle` — нейтральный серый, без анимации.
  - `working` — пульсирующий индикатор + короткий caption под кубиком с именем текущего тула (если выводимо из активной сессии).
  - `awaiting_user` — индикатор «требуется ответ пользователя» (как сейчас в #0024 — сохранить визуальный язык).
- Состояние выводится из live-state поллинга (#0024): активные сессии рана + статус ран-файла. Алгоритм:
  - Есть активная сессия, в которой роль — участник, и последнее сообщение **не** от этой роли → `working`.
  - Есть активная сессия, последнее сообщение от этой роли и адресовано user → `awaiting_user` (только для product, как сейчас).
  - Иначе → `idle`.
- Только индикаторы перерисовываются на каждом poll-tick. Линии-«репортинги» (из #0042) — нет.
- Storybook-story: канвас с тремя кубиками в разных состояниях.
- Unit на функцию `cubeStateFor(role, runState): 'idle'|'working'|'awaiting_user'`.

## Implementation notes

- Caption «имя тула» — опциональный, если из live-state не выводится — пропустить, не заглушки.
- Не делать нового статуса в run-файле — статус кубика чисто derived.
