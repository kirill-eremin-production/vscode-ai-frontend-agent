## Context

После #0015 в проекте есть токены и Tailwind, но всё ещё нет единых атомов: каждый компонент вёрстает кнопки/спиннеры/бейджи самостоятельно. Это:

1. делает следующие задачи (чат-bubble #0020, tool-карточки #0021, canvas #0023) дороже — каждая повторяет базовое;
2. гарантирует визуальную несогласованность по мере роста UI.

Решение — собрать минимальную библиотеку атомов в `src/webview/shared/ui/`, прикрыть её Storybook'ом для разработки в изоляции и зафиксировать как единственный источник этих примитивов.

Storybook — именно для разработки/документации компонентов; визуальные регресс-тесты в Playwright откладываем до отдельной задачи (компоненты должны устаканиться).

## Acceptance criteria

Состав библиотеки (`src/webview/shared/ui/`) — каждый атом со своим `.stories.tsx`:

- **Button** — варианты `primary | secondary | ghost | danger`, размеры `sm | md`, опциональная иконка слева/справа, состояние `loading` (внутри спиннер из ниже), `disabled`. Использует vscode-button-токены.
- **IconButton** — квадратная кнопка с иконкой, варианты как у Button + `aria-label` обязателен (lint-rule).
- **Spinner** — крутящийся индикатор, размеры `xs | sm | md`, цвет наследует `currentColor`. Без подписи.
- **LoadingState** — композиция: Spinner + подпись справа («Архитектор думает…»). Используется в шапках/кнопках/канвасе для US-23.
- **Skeleton** — серый плейсхолдер с пульсацией, варианты `text | line | block`. Для пустых состояний загрузки.
- **Badge** — компактный пилюль-индикатор, варианты `neutral | accent | danger | warning | success`. Используется для статусов и `kind` сессий.
- **Panel** — карточка с бордером и фоном `surface-elevated`, опциональный `header` slot.
- **Collapsible** — обёртка с управляемым/неуправляемым раскрытием, иконка-шеврон, плавный transition. Используется в tool-карточках #0021.
- **EmptyState** — иллюстрация (lucide-иконка большого размера) + заголовок + описание + опциональный CTA. Для empty-state в main-area #0018.
- **Avatar** — круг/квадрат с иконкой роли (lucide), цвет фона по `--color-role-*`. Размеры `sm | md | lg`. Используется в чат-bubble #0020 и на канвасе #0023.
- **Tooltip** — поверх `@radix-ui/react-tooltip` (или Floating UI — что легче по зависимостям). Срабатывает по hover/focus, поддержка клавиатуры.

Storybook:

- Storybook 8 (Vite-builder), сторис в `src/webview/shared/ui/**/*.stories.tsx` рядом с компонентами.
- В preview подключены те же CSS-токены из #0015 (импорт `app.css` или эквивалента) — компоненты выглядят так же, как в боевом webview.
- Toolbar-аддон для переключения темы VS Code (light/dark/high-contrast) через подмену CSS-переменных на корне — чтобы в одной сторис проверять три темы.
- `npm run storybook` поднимает её локально; сборка `npm run build-storybook` собирается без ошибок.
- README раздел «Компонентная библиотека» в [src/webview/shared/ui/README.md](src/webview/shared/ui/README.md) — что есть, как добавлять новый атом, почему мы держим это в одном месте.

Качество:

- Каждый атом — чистая функция от пропсов (нет глобального state). Тёмно-светлая тема — только через токены.
- Атомы не тянут за собой бизнес-логику (никаких `useRunsStore` внутри Button'а).
- Все publicly-exported атомы реэкспортятся из `src/webview/shared/ui/index.ts` для удобного импорта `import { Button, Spinner } from '@webview/shared/ui'`.

## Implementation notes

- Базовые tooltip/popover — `@radix-ui/react-tooltip`/`@radix-ui/react-popover`. У Radix хорошая accessibility, и они не тянут стилей — оформление будет нашими Tailwind-классами.
- `@radix-ui/react-collapsible` — для Collapsible (избавляет от ручной работы с aria/анимациями).
- Иконки — `lucide-react` (уже добавлен в #0015). Маппинг ролей к иконкам — отдельный модуль `src/webview/shared/ui/role-icons.ts` (Product → `Package`, Architect → `Compass`, User → `User`, System → `Cog`). Канвас и Avatar используют этот же маппинг.
- Для Storybook-аддона переключения тем — простой decorator, который ставит `data-theme` на `body` и переопределяет несколько CSS-переменных (мокаем vscode-переменные на демо-значения для каждой темы).
- Visual regression — отдельный будущий тикет; здесь не делать. Но структура сторис готова к подключению (`tags: ['autodocs', 'visual']`).
- Менять существующие компоненты на новые атомы в этой задаче не нужно (миграция — в #0017–#0022). Тут только создание библиотеки и стори.
