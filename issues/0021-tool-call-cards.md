## Context

Сейчас вызовы инструментов рендерятся в ленте как однообразные блоки текста (имя + JSON), визуально неотличимы от чат-сообщений. Это нарушает US-11 в части «понятно глазами, что происходит» и плохо масштабируется на цепочку из десятка `kb.read`'ов.

Нужны компактные карточки со статусом, иконкой инструмента и раскрываемыми деталями. Tool-карточки визуально отличаются от чат-bubble'ов (#0020), чтобы пользователь моментально различал «сказал агент» и «вызвал инструмент».

## Acceptance criteria

- Новый компонент `src/webview/features/chat/ui/ToolCard.tsx`:
  - **Свёрнутый вид** (по умолчанию): одна строка — иконка инструмента (lucide, маппинг ниже), имя тула моноширинным шрифтом, бейдж статуса (`running` с пульсирующей точкой / `ok` зелёный / `error` красный), однострочный summary аргументов (например, `path: "features/dark-theme"`), время и длительность справа.
  - **Развёрнутый вид** (Collapsible #0016): полные input/output JSON блоками с моноширинным шрифтом и vscode-token-цветами. Для `error` — текст ошибки сверху, на видном месте.
  - Карточка визуально отличается от ChatMessage: другой фон (более нейтральный), другая форма (компактнее, без аватара слева), thin-бордер.
- Маппинг тула → иконка (lucide), в `src/webview/shared/ui/tool-icons.ts`:
  - `kb.list` → `List`
  - `kb.read` → `FileText`
  - `kb.grep` → `Search`
  - `kb.write` → `FilePlus`
  - `ask_user` → `MessageCircleQuestion`
  - неизвестный тул → `Wrench` (fallback)
- Состояние «развёрнут/свёрнут» — локальное per-карточка. По умолчанию свёрнуты **все**, кроме:
  - последней карточки со статусом `error` в текущей видимой сессии — раскрыта автоматически (ошибки не должны прятаться).
- Ссылки на файлы внутри карточки (US-11): пути аргументов `kb.write.path`/`kb.read.path` рендерятся как кликабельные. Клик отправляет IPC `workspace.openFile { path }` (если такого ещё нет — добавить тонкий хэндлер). Поведение — как сейчас, только обёрнуто в новый UI.
- Длинные JSON блоки — с горизонтальным скроллом, не ломают вёрстку. Лимит на rendered-длину input/output (например, 4 KB) с пометкой «… показано первые 4 KB, открыть полностью» (опционально — оставить заглушку, если в текущих ранах редко > 4 KB).
- Лента в `ChatFeed.tsx` (#0020) выбирает рендерер по типу записи: `ChatMessage` для чат-сообщений, `ToolCard` для tool-events. Хронологическая склейка остаётся как сейчас.

## Implementation notes

- `Collapsible` (#0016) — основа карточки. Анимация раскрытия — плавная, не дёрганая.
- JSON-подсветка — без полноценного highlighter'а (`shiki`/`prismjs` слишком тяжёлые). Достаточно `<pre>` с моноширинным шрифтом и подцветить ключи vs значения простым regex'ом, либо вообще оставить плоским моноширинным — проверить визуал глазами и решить. На этом этапе экономим на зависимостях.
- Длительность — `endedAt - startedAt`, если оба есть в `ToolEvent`. Если ещё бежит — показываем «N сек» на основе `startedAt` и текущего времени, обновляем через `useEffect` + `setInterval(1000)` пока статус `running`.
- Аккуратно с auto-expand error: триггер только на «последней error-карточке в видимой сессии», иначе весь ран превратится в стену развёрнутых ошибок.
- TC: добавить мини-проверку — после успешного `kb.list` карточка свёрнута, после неуспешного — развёрнута; клик по шевронy раскрывает/сворачивает.
