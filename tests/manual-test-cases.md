# Manual test cases

Шаги предполагают запущенный Extension Development Host (`F5`) с открытой произвольной папкой-«песочницей». Перед каждым кейсом ОБЯЗАТЕЛЬНО фиксируй стартовое состояние: что лежит в `.agents/`, задан ли ключ.

> **Подсказка по проверке секретов.** В Extension Development Host открой палитру (`Cmd+Shift+P`) → `AI Frontend Agent: Set OpenRouter API Key`. Чтобы «забыть» ключ — введи пустую строку (это удалит запись).

---

## TC-1. Установка ключа OpenRouter через команду — US-1

1. Палитра → `AI Frontend Agent: Set OpenRouter API Key`.
2. Ввести валидный ключ `sk-or-...`, Enter.

**Ожидание:**

- Уведомление «OpenRouter API key сохранён».
- В Activity Bar открыть AI Frontend Agent — кнопка «Set API key» при создании рана больше не вызывает input box (если ключ ещё не запрошен — кейс TC-2 проверит это).

## TC-2. Установка ключа через кнопку «Set API key» — US-1

1. Удалить ключ (см. подсказку выше).
2. Открыть сайдбар расширения.
3. Нажать кнопку «Set API key» под textarea.

**Ожидание:** появляется тот же input box. После сохранения дальнейшие запуски ранов идут без повторного вопроса.

## TC-3. Создание первого рана — US-2

Стартовое состояние: ключ задан, `.agents/` отсутствует или пуст.

1. В textarea ввести `Сделать кнопку логина с тёмной темой`.
2. Нажать «Start run».

**Ожидание:**

- В файловой системе появилась `.agents/runs/<runId>/` с `meta.json` и `chat.jsonl`.
- `meta.json` содержит `id`, `title` (3–6 слов, по-русски), `prompt` (исходный текст), `status: "draft"`, `createdAt`, `updatedAt`.
- `chat.jsonl` содержит ровно одну строку с JSON: `{from: "user", text: "Сделать кнопку логина...", ...}`.
- В UI слева новый ран в начале списка, выбран, справа — карточка деталей с этим promptом.
- Textarea очищена.

## TC-4. Создание рана при недоступной модели заголовка — US-2 (fallback)

Имитация: задать заведомо неверный ключ (любая строка) либо отключить интернет.

1. Ввести запрос `Test fallback`, Start run.

**Ожидание:**

- Ран всё равно создан.
- `meta.title` равен исходному запросу (или его первым 40 символам + `…`), а не пустая строка.
- В UI ошибка не показывается (заголовок просто менее красивый).

## TC-5. Список ранов после перезапуска — US-3

1. Создать 2–3 рана подряд с разными заголовками.
2. Закрыть Extension Development Host.
3. Запустить заново (`F5`), открыть ту же папку.

**Ожидание:** список содержит те же раны, свежий — сверху. Даты и статусы совпадают с тем, что было до перезапуска.

## TC-6. Переключение между ранами — US-4

1. Создать минимум 2 рана.
2. Кликать по разным элементам списка.

**Ожидание:**

- Подсветка выбранного элемента переключается мгновенно.
- Карточка справа сразу заменяется (на время загрузки может быть «Загружаю…»).
- Заголовок, prompt и счётчик сообщений соответствуют выбранному рану.

## TC-7. Удаление папки рана вручную — US-4 (отказоустойчивость)

1. Создать ран, выбрать его.
2. В файловой системе удалить `.agents/runs/<runId>/`.
3. В UI кликнуть «обновить» косвенно — например, нажать на любой другой ран и обратно.

**Ожидание:**

- Расширение не падает.
- При попытке выбрать удалённый ран показывается «Загружаю…» бесконечно (это допустимо на текущей итерации) или пустая карточка — главное, нет необработанных исключений в Output → AI Frontend Agent.
- При следующем `runs.list` (перезапуск или новый ран) удалённый исчезает из списка.

## TC-8. Запуск без открытой папки — US-5

1. Закрыть все папки в Extension Development Host (File → Close Folder).
2. Открыть сайдбар расширения, ввести любой prompt, Start run.

**Ожидание:**

- Появляется ошибка-уведомление: «Откройте папку в VS Code, чтобы создавать раны (нужно место для .agents/)».
- На диске ничего не создано.

## TC-9. Отмена ввода ключа — US-5

1. Удалить ключ.
2. В сайдбаре ввести prompt, Start run.
3. В появившемся input box нажать `Esc` (или закрыть).

**Ожидание:**

- Ран не создан, ошибки в UI нет.
- Список ранов не изменился.

## TC-10. Открытие в полноразмерной вкладке — US-6

1. В сайдбаре нажать «Open in tab».
2. В открывшейся вкладке создать ещё один ран.
3. Снова нажать на иконку сайдбара.

**Ожидание:**

- В сайдбаре после повторного открытия (или `runs.list` от его монтирования) виден ран, созданный из вкладки.
- Повторное «Open in tab» из сайдбара не открывает вторую вкладку, а фокусирует существующую.

## TC-11. Tool-loop: успешный цикл с одним kb.write

Стартовое состояние: ключ задан, доступна тестовая команда `AI Frontend Agent: Run Tool Loop Smoke` (palette).

1. Палитра → `AI Frontend Agent: Run Tool Loop Smoke`.
2. Ввести prompt вида `Создай в kb файл note.md с текстом "hello"`.

**Ожидание:**

- В `.agents/runs/<id>/tools.jsonl` появились записи: минимум одна `assistant` с `tool_calls`, одна `tool_result` (успех `kb.write`), финальная `assistant` без `tool_calls`.
- В `.agents/knowledge/product/note.md` (или соответствующий путь, который подскажет промпт) лежит файл с ожидаемым содержимым.
- В `chat.jsonl` появилась только финальная reply агента — внутренние tool_calls туда **не дублируются**.
- Статус рана: `running → awaiting_human` (для smoke — финальный «успех», approve-кнопок нет).

## TC-12. Tool-loop: sandbox-нарушение в kb.read

1. Запустить smoke-цикл с prompt'ом, который явно подталкивает агента вызвать `kb.read({ path: "../../../etc/passwd" })` (или поправить prompt вручную через debug).

**Ожидание:**

- Tool-result содержит `{ error: "..." }` с упоминанием sandbox.
- Файл за пределами `.agents/knowledge/` **не прочитан**.
- Цикл не падает: модель видит ошибку, может попробовать другой путь или завершить ран. В `tools.jsonl` ошибка зафиксирована.

## TC-13. Tool-loop: невалидные входы тула

1. Запустить smoke-цикл с prompt'ом, провоцирующим невалидный вызов (например, `kb.write` без поля `content`).

**Ожидание:**

- Ajv-валидация вернула ошибку как `tool_result.error` (тул-handler **не вызывался**).
- Модель получает ошибку и может пере-вызвать тул правильно.
- Ничего не записано в `.agents/knowledge/`.

## TC-14. Tool-loop: превышение лимита итераций

1. Запустить smoke-цикл с prompt'ом, провоцирующим бесконечный цикл tool-вызовов (например, «бесконечно читай и переименовывай»).

**Ожидание:**

- После N=20 итераций ран помечен `failed`.
- В `tools.jsonl` есть финальная запись с описанием причины (`iteration limit exceeded`).
- В `chat.jsonl` — понятное сообщение об ошибке от `agent:system` (или аналогичного источника).
- На диске остаются все промежуточные артефакты — для разбора.
