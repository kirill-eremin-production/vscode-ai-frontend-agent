# Тестовые артефакты

В этой папке лежат материалы для ручной проверки расширения. Автоматизированных тестов пока нет — на текущем этапе важнее видеть весь сценарий руками в Extension Development Host (F5), чем закрывать его юнитами.

## Что здесь

- [user-stories.md](user-stories.md) — пользовательские истории, описывающие, ЧТО и ЗАЧЕМ делает пользователь.
- [e2e/specs/](e2e/specs/) — каждый тест-кейс лежит отдельным файлом:
  - `tc-N-...spec.ts` — автоматизированный e2e (Playwright + настоящий VS Code), описание кейса в комментарии-блоке самого файла.
  - `tc-N-...md` — ручной кейс (e2e-автоматизации пока нет), шаги и ожидаемый результат внутри markdown'а.
  - Имя файла = единственная точка нумерации (`TC-1`, `TC-12`). Удалённые номера не переиспользуем.
- [checklist.md](checklist.md) — короткий чек-лист для smoke-проверки перед коммитом.

## Как запускать

1. `nvm use && npm install && npm run build`
2. В VS Code открыть этот репозиторий и нажать `F5` — поднимется Extension Development Host.
3. **Важно:** в окне-хосте ОТКРЫТЬ КАКУЮ-ТО ПАПКУ (любую) — без неё расширение не сможет писать в `.agents/`.
4. Иконка «AI Frontend Agent» появится в Activity Bar.

## Артефакты на диске

После прохождения сценариев в `<workspace>/.agents/runs/` останутся папки ранов. Их можно удалять руками; расширение не падает на отсутствующих/повреждённых файлах.

## Когда писать новый тест-кейс

При добавлении новой роли, инструмента или сообщения в IPC — сначала описывай user story, потом заводи новый файл в [e2e/specs/](e2e/specs/) (`.md` — если автоматизировать пока не получается, `.spec.ts` — если можем). Сначала контракт — потом код.
