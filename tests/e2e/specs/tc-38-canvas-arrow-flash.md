# TC-38. Канвас — анимация стрелки при появлении user-вмешательства

US-26 (#0025). Когда пользователь вмешивается в bridge (TC-32), на канвасе появляется новая стрелка `user->architect`. Эта стрелка должна **анимироваться** при появлении: атрибут `data-arrow-flashing="true"` стоит первые ~3 секунды, затем снимается.

Этот TC ловит:

- если `useArrowFlashes` перестанет диффить layout edges (новые edge'ы не зафлэшат) — атрибут не появится никогда;
- если перестанет работать «историческая глухота» (флэш сработает на первый mount канваса) — пройдёт другой тест (TC-37 не ждёт `data-arrow-flashing`);
- если сломается setTimeout-сброс — атрибут не уйдёт через 3 секунды (deselect flag будет торчать).

## Шаги

1. Сценарий hybrid (как TC-32): title → продактовый kb.list → BRIEF → архитекторский kb.list → PLAN_V1 → architect_list_v2 → PLAN_V2.
2. Создать ран, дождаться `plan.md` v1 и `awaiting_human`.
3. Открыть webview, выбрать ран. Канвас по умолчанию.
4. Зафиксировать: до user-вмешательства атрибута `data-arrow-flashing="true"` нет ни на одной стрелке (историческая глухота).
5. Отправить user-followup через composer (composer работает поверх канваса, см. #0023).
6. В течение ~5 секунд должна появиться стрелка `[data-canvas-edge="user->architect"][data-arrow-flashing="true"]`.
7. Дождаться `plan.md = v2` (как в TC-32).
8. Через ~5 секунд после появления флэша атрибут `data-arrow-flashing` должен сняться (анимация ~3 сек на kind='appear').

## Ожидание

Флэш появляется → задерживается ≥3 сек → исчезает. Стрелка остаётся на канвасе (без атрибута).
