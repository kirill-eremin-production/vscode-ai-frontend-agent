# TC-37. Канвас — живые статусы агентов после handoff'а

US-25 (#0024). После того, как продакт сдал бриф и архитектор автоматически отработал план, на канвасе:

- кубик архитектора имеет `data-canvas-activity="awaiting_human"` и подпись «Готово» (роль владеет активной сессией, статус `awaiting_human`);
- кубик продакта переходит в `data-canvas-activity="idle"` с подписью «закончил бриф» (handoff случился — продакт больше не владеет активной сессией);
- одновременно ни на одном кубике нет `thinking`/`tool` (нет «двух работающих» — acceptance issue);
- user-кубика на канвасе нет (без вмешательства).

Этот TC ловит:

- если `selectActiveSessionForRole` сломается (не найдёт сессию роли) — кубик станет `idle` с прочерком, не «закончил бриф»;
- если `ownerRoleOfActiveSession` начнёт возвращать неправильную роль — оба кубика покажут одинаковый статус (или у обоих будет `awaiting_human`);
- если канвас перестанет реактивно обновляться по `runs.updated` — кубики останутся в старом статусе (`thinking` или `idle` с пустой подписью).

## Шаги

1. Сценарий handoff'а (как TC-31): title → продактовый kb.list → brief → архитекторский kb.list → plan, `AUTOSTART_ARCHITECT=1`.
2. Создать ран, дождаться `plan.md` и `awaiting_human`.
3. Открыть webview, выбрать ран. Дефолтная вкладка — «Карта».
4. Проверить:
   - `[data-canvas-role="architect"][data-canvas-activity="awaiting_human"]` — есть;
   - `[data-canvas-role="product"][data-canvas-activity="idle"]` — есть;
   - подпись на продакте содержит «закончил бриф»;
   - на канвасе нет ни одной ноды с `data-canvas-activity="thinking"` или `data-canvas-activity="tool"` (нет «двух работающих»);
   - user-кубика нет.

## Ожидание

Канвас отражает живой статус ровно одного владельца активной сессии; остальные роли — idle с подписью артефакта.
