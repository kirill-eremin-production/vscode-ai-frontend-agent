# TC-34. Канвас — дефолтная вкладка и одиночный продакт

US-24 (#0023). После выбора рана main-area по умолчанию открывается на вкладке **«Карта»**, а не на ленте чата. На пустом ране (или ране, где работал только продакт) канвас рендерит ровно один кубик роли `product`. Чат остаётся доступен второй вкладкой; переключение работает в обе стороны.

Этот TC ловит самые дешёвые регрессии:

- если кто-то поменяет дефолт обратно на `chat` — мы это увидим;
- если канвас перестанет рендерить ноду роли — `data-canvas-role="product"` исчезнет;
- если переключатель «Карта/Чат» сломается (выпадет `aria-selected`, перестанет триггерить смену контента) — упадёт переключение в TC.

## Шаги

1. Поднять окружение и smoke-сценарий продакта без handoff (как в TC-17): пустой `kb.list`, затем финальный бриф.
2. Создать ран через UI, дождаться `brief.md` и статуса `awaiting_human`.
3. Открыть webview, выбрать ран в списке.
4. **Не делать** ручного переключения вкладок.
5. Проверить, что:
   - вкладка «Карта» имеет `aria-selected="true"`;
   - в DOM присутствует `[data-canvas-root]` со вложенным `[data-canvas-role="product"]`;
   - других ролей-кубиков на канвасе нет (в частности, нет `[data-canvas-role="architect"]`);
   - рёбер handoff'а нет.
6. Кликнуть вкладку «Чат», убедиться, что появляется `.run-details__composer-input` и tool-карточка `kb.list` (`.tool-card[data-tool="kb.list"]`), а кубики канваса исчезают.
7. Кликнуть «Карта» обратно — кубик продакта снова виден, tool-карточек нет в DOM.

## Ожидание

Все шаги проходят без ошибок. Дефолтная вкладка — карта; single-agent ран всегда отрисовывается одной нодой `product`.
