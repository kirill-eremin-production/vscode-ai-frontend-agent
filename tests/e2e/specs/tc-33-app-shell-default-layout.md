# TC-33. Трёхпанельный shell — дефолт и расположение «+ Новый ран» — #0017

Стартовое состояние: чистый профиль VS Code (нет сохранённых UI-префов
в `globalState`), окно расширения открыто как полноразмерная панель
(шире 700px).

1. Открыть webview через команду `AI Frontend Agent: Open in Tab`.

**Ожидание (дефолтное состояние):**

- Видны три колонки: левый сайдбар «Раны», центральная main-area,
  правый сайдбар «Сессии».
- Обе боковые панели — в раскрытом состоянии (полная ширина,
  виден заголовок и контент). Кнопки «Свернуть» с шевроном видны.
- В шапке левой панели присутствует IconButton «Новый ран» (icon `Plus`)
  — рядом с кнопкой «Свернуть». Внутри тела списка ранов **нет**
  composer'а / textarea / кнопки «Start run».
- Правая панель показывает заглушку «Sessions panel — TBD #0019».

2. Кликнуть «Свернуть» на левой панели.

**Ожидание:**

- Левая колонка сжимается до 32px полосы, в которой видны только два
  IconButton'а: «Развернуть» (icon `ChevronRight`) и «Новый ран».
- Main-area автоматически расширяется на освободившееся место.

3. Перезапустить VS Code (Reload Window) и снова открыть webview.

**Ожидание:**

- Левая панель остаётся свёрнутой (состояние сохранилось через UI-преф
  `shell.leftPanelCollapsed` в `globalState`).

4. На узком окне (ширина < 700px) в сценарии «sidebar-view с
   нетронутыми UI-префами»:

**Ожидание:**

- При первом старте обе панели свёрнуты по умолчанию (порог
  `NARROW_WINDOW_PX`); main-area занимает всю ширину.
- Развёртывание любой панели вручную сохраняется и переживает
  перезапуск.

5. Кликнуть «Новый ран» (любая из позиций — раскрытая шапка или
   collapsed-полоса).

**Ожидание:**

- Main-area переключается в режим `'new-run'` и показывает заглушку
  «Форма создания — #0018» (реальная форма появится в #0018).
- Клик по любому рану в списке возвращает main-area в режим
  `'run-details'` с карточкой выбранного.
