# TC-39. Канвас — drill-in в чат сессии (US-27, #0026)

Клик по кубику или стрелке на канвасе — это «провалиться» в чат соответствующей сессии: main-area переключается на вкладку «Чат», в правой панели выбирается нужная сессия, последняя выбранная сессия запоминается per-run и восстанавливается при ручном возврате на «Чат».

## Что ловим

- Если drill-in не сработает — на канвасе клик не приведёт к смене вкладки или сессии (регрессия в `RunCanvas.onDrillIn` / `drillIntoSession`).
- Если потеряется `bridgeSessionId` на стрелке — клик по handoff-стрелке откроет произвольную (как правило, активную) сессию, а не bridge — поломается `data-canvas-edge-session`.
- Если `lastViewedSessionByRun` не сохранится — повторный клик «Карта → Чат» вернёт пользователя на активную сессию вместо последней просмотренной.

## Шаги

1. Сценарий handoff (как TC-37): title → product kb.list → BRIEF → architect kb.list → PLAN. Архитектор включён через `AUTOSTART_ARCHITECT=1`.
2. Создать ран, дождаться `awaiting_human` и `plan.md`.
3. Открыть webview, выбрать ран. Канвас по умолчанию.
4. Зафиксировать у handoff-стрелки `data-canvas-edge-session` — это id bridge-сессии (architect).
5. Кликнуть по кубику `architect`:
   - вкладка `chat` стала `aria-selected=true`,
   - в правой панели выделена architect-сессия (она же bridge — у архитектора нет другой).
6. Вернуться на «Карту» → кликнуть по handoff-стрелке `product->architect`:
   - вкладка снова `chat`,
   - выделена та же bridge-сессия.
7. Кликнуть по кубику `product`:
   - выделена product-root-сессия (а не bridge).
8. Вернуться на «Карту», ничего не меняя; снова кликнуть кнопку «Чат» в табах:
   - выделена product-root-сессия (последняя viewed-сессия восстановилась).

## Ожидание

Каждый клик по элементу канваса немедленно (≤ 2с) переключает вкладку на «Чат» и подсвечивает соответствующую сессию в правой панели. Возврат на «Чат» через таб поверх «Карты» восстанавливает последнюю просмотренную сессию.
