# TC-49. Canvas рисует иерархию ролей без стрелок коммуникации

Покрывает US-42 (#0042). Ручной smoke: визуально убеждаемся, что
canvas стал org-chart'ом — кубики ролей по уровням, тонкие линии
между ними, без анимаций и стрелок коммуникации.

## Предусловия

- VS Code открыт в чистом workspace, OpenRouter-ключ задан.
- Расширение собрано (`npm run build`).

## Шаги

1. Создать новый ран с любым промптом, дождаться, пока отработают
   три роли — продакт, архитектор и программист (можно использовать
   тот же сценарий, что в TC-32 multi-agent-hybrid).
2. Перейти на вкладку «Карта команды» (canvas).
3. Визуально проверить раскладку:
   - Три кубика расположены **строго друг над другом** по вертикали,
     все выровнены по центру по горизонтали (одинаковый `x`).
   - Порядок сверху вниз: **Продакт → Архитектор → Программист**.
   - Между кубиками — **две тонкие статичные линии** без стрелок,
     без подписей, без бегающего пунктира.
4. В DevTools webview'а (Help → Toggle Developer Tools) открыть
   Inspector и убедиться:
   - У каждого кубика есть атрибут `data-canvas-role` (значение —
     роль) и `data-canvas-level` (значение — индекс роли в иерархии:
     0 для продакта, 1 для архитектора, 2 для программиста).
   - У каждого кубика есть атрибут `data-canvas-drill-session` со
     значением — id сессии, которая откроется при клике (drill-in).
   - Линии между уровнями имеют атрибут
     `data-canvas-reporting-line` со значением `product--architect`
     и `architect--programmer`. Стрелок (`marker-end`) у них **нет**.
   - В DOM **отсутствуют** атрибуты `data-canvas-edge`,
     `data-canvas-edge-kind`, `data-arrow-flashing`,
     `data-arrow-flash-kind` и классы `run-canvas__edge--*`.
5. Кликнуть по кубику архитектора. Должна открыться вкладка «Чат»
   с сессией архитектора (drill-in работает). Вернуться на canvas.
6. Прислать продакту новое сообщение (через тот же канал, что в
   TC-32) и убедиться: между кубиками **не** появляется бегающий
   пунктир / пульсация / любая другая анимация edge'а. Линии
   остаются такими же, как до сообщения.
7. Попробовать перетащить любой кубик — он **не** должен
   перетаскиваться (DnD отсутствует). Pan/zoom фоном (drag по
   пустому месту, колесо мыши, клавиша `0`) — продолжают работать.

## Ожидаемый результат

Все шаги 3–7 выполняются без отклонений. На canvas нет ни одной
стрелки коммуникации и ни одной анимации edge'ов. Drill-in по
кубикам работает.

## Известные ограничения

- Кубик пользователя (`user`) на canvas пока не рисуется — он
  появится в #0043 (US-43, отдельная задача).
- Состояния кубика (paused / awaiting_meeting и т.п.) появятся в
  #0044.
- Старые TC-35 / TC-36 / TC-38 (handoff-стрелки и flash-анимация)
  становятся неактуальными после #0042 — будут переработаны в
  рамках #0045.
